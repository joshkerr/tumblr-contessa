Nice Tumblr workflow for the ruby user

I’ve got a new workflow to submit posts to Tumblr. When I’m not using Cortex, I’m using iA Writer and a ruby script to send my content to Tumblr without even opening a web browser.

Cortex is great for resubmitting other user’s content via the web, but when it comes to writing your own articles, forget it. This is where iA Writer comes in. This app is designed for writing, it allows you to focus on the words (think full screen editor) and it has great markdown support.

You can’t easily submit articles from a text editor (unless you’ve rigged up support in Textmate) so you need to setup a script to send your posts for you. Here is mine:

#!/bin/bash
IFS=$(echo -en "\n\b")
cd "~/Dropbox/Blog/BlogArticles/"
set `ls -t -r | tail -n 1`
echo "$1"
tumblr -c ~/.tumblrlogin "$1"

I’m using a ruby gem: —tumblr-rb— and have it setup with my Tumblr blogs username and password in my ~/.tumblrlogin file. More on the gem’s man pages about configuring the gem.

The script is simple. It looks for the most recent post in my blog article directory and submits it. In my post, I’ve got YAML config items that allow me to control where the post goes: queue, drafts, published, etc..

title: post title
type: regular
format: markdown
state: published
tags: tag1,tag2,tag3

So I write and save the posts in my blog directory and then execute my script to send them up to Tumblr. If I wanted I could rig the script to open the browser and go right to the post (the ruby gem returns the post ID after a successful post.)

I could do a lot more with it, but for now it is getting the job done.
